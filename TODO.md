# Ayah Studio - TODO

## الاتجاه الحالي للبرنامج

Ayah Studio الآن هو مولد فيديوهات قرآن عمودية Web Mobile لحساب `@tilawat_alquran30`.
النسخة الحالية لا تعتمد على خطة رفع فيديو عشوائي والتعرف عليه كمسار رئيسي، بل تعتمد على:

1. اختيار السورة والآيات مباشرة.
2. اختيار القارئ.
3. اختيار خلفية مناسبة أو مقترحة.
4. معاينة الفيديو داخل إطار هاتف 9:16.
5. توليد MP4 عبر backend محلي.

هذا الملف هو المرجع الحالي للعمل. أي بنود قديمة عن prototype سابق أو TODO طويل قديم تعتبر تاريخية وليست خطة التنفيذ المعتمدة الآن.

- الواجهة تعرض لوحة حالة صغيرة لطلب التوليد الحالي، تشمل رقم الطلب والحالة والتقدم التقريبي، حتى تكون تجربة الهاتف أوضح أثناء الانتظار.
- الشريط الثابت في الهاتف يستخدم زر توليد سريع، وبعد اكتمال التوليد يتحول إلى حالة `الفيديو جاهز` ويعرض أزرار فتح وتحميل وإرسال ونسخ الوصف.
- أخطاء التوليد المصنفة مثل FFmpeg والخط والخلفية وصوت القارئ تمر الآن إلى الواجهة عبر `code` و`hint` لعرض سبب مفهوم للمستخدم.
- سياسة الصوت مثبتة باختبار `scripts/test-recitation-audio-policy.mjs`: تحميل آية-بآية حسب القارئ والسورة والآية، بدون تغيير سرعة أو طبقة التلاوة.
- سياسة نص القرآن مثبتة باختبار `scripts/test-quran-text-policy.mjs`: النص يأتي من `data/quran-uthmani.json` ولا توجد واجهة لتعديله يدوياً.

## ملاحظات مهمة

- لا يتم توليد نص القرآن بالذكاء الاصطناعي.
- لا يتم تعديل نص الآية يدوياً كأنه نص عادي.
- لا يعتمد النشر النهائي على التعرف الآلي وحده.
- لا يتم تغيير سرعة التلاوة أو طبقة الصوت.
- الخلفيات يجب أن تكون مناسبة ومحترمة ولا تضعف قراءة الآيات.
- أي فيديو نهائي يحتاج مراجعة بشرية قبل النشر.
