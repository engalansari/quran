# Ayah Studio

Ayah Studio is a Web Mobile Quran video generator for `@tilawat_alquran30`.

The current version is based on explicit selection, not automatic recognition:

1. Choose the surah and ayahs.
2. Choose the reciter.
3. Choose a suitable background.
4. Preview the 9:16 phone frame.
5. Generate an MP4 through the local backend.

The Quran text is loaded from `data/quran-uthmani.json`. It is not generated by AI.

## Current Flow

Run the local backend:

```powershell
npm.cmd start
```

Open:

```text
http://127.0.0.1:4173/
```

The main generate button uses:

```text
POST /api/render
GET /api/render/:jobId
```

Render jobs move through:

```text
pending -> processing -> completed | failed
```

Generated videos are written under `outputs/`.

## Phone Testing

For the connected Android phone, use the phone helper script:

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File scripts\start-phone-server.ps1 -Port 4175
```

Then run:

```powershell
adb reverse tcp:4175 tcp:4175
adb shell am start -a android.intent.action.VIEW -d "http://127.0.0.1:4175/"
```

The tested phone flow:

- Open Ayah Studio on the phone.
- Press `توليد الفيديو` from the fixed bottom bar.
- Wait for MP4 generation.
- The bottom bar changes to `الفيديو جاهز`.
- Actions appear: `فتح`, `تحميل`, `إرسال`, `نسخ الوصف`.

## External Audio Mode

Instagram link scraping is postponed.

The current external mode supports uploading an audio or video file from the device, then generating a new MP4 with the selected background. Reciter name and ayah metadata can be optional in that mode.

## Online Foundation

The repository includes initial online deployment scaffolding:

- `firebase.json`
- `.firebaserc.example`
- `firestore.rules`
- `storage.rules`
- `Dockerfile`
- `.dockerignore`
- `FIREBASE.md`

This is not deployed yet. The intended direction is Firebase Hosting for the web UI and Cloud Run for FFmpeg/backend rendering.

Before deployment, create `.firebaserc` from `.firebaserc.example`, set the real project id, then run:

```powershell
npm.cmd run proof:firebase-ready
```

See `FIREBASE.md` for the full deployment sequence.

## Important Limits

- Final publishing still needs human review.
- Quran text must remain sourced from reviewed local data.
- The app must not alter Quran wording.
- The app must not change recitation speed or pitch.
- Backgrounds must be respectful and readable.
- Instagram API posting/scheduling is a later phase.
- Firebase/Cloud Run online operation is scaffolded but not production-ready yet.
